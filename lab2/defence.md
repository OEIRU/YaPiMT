# Ответы на вопросы по лабораторной работе №1

## 1. Постоянные таблицы, используемые в трансляторах

Постоянные таблицы создаются одновременно с проектированием транслятора и остаются неизменными в процессе работы с исходной программой. Они содержат заранее определенные данные, необходимые для корректной работы компилятора. К таким таблицам относятся:

- **Таблицы алфавита входного языка**: содержат символы, разрешенные в языке программирования (например, буквы, цифры, специальные символы).
- **Таблица зарезервированных слов**: хранит ключевые слова языка (например, `while`, `if`, `main`), которые имеют специальное значение и не могут использоваться как идентификаторы.
- **Таблицы методов синтаксического анализа**: используются для реализации алгоритмов разбора (например, LL- или LR-разбор).

Эти таблицы обычно упорядочены, что позволяет эффективно выполнять поиск элементов.

---

## 2. Переменные таблицы, используемые в трансляторах

Переменные таблицы создаются динамически в процессе трансляции исходной программы. Они хранят информацию, которая становится известной только во время анализа программы. Основные переменные таблицы:

- **Таблица символов (идентификаторов)**: содержит записи об идентификаторах (имена переменных, функций и т.д.) с их атрибутами (тип, область видимости, память и т.д.).
- **Таблица констант**: хранит значения констант, встречающихся в программе, с указанием их типов (например, числовые или строковые константы).

Эти таблицы могут динамически расти и изменяться в зависимости от структуры исходной программы.

---

## 3. Операции с таблицами

Основные операции, выполняемые с таблицами в трансляторах:

- **Поиск элемента**: проверка наличия лексемы или атрибута в таблице.
- **Добавление элемента**: вставка новой записи (например, нового идентификатора или константы).
- **Обновление атрибутов**: изменение информации о существующей записи (например, добавление типа или области видимости).
- **Удаление элемента**: удаление записи из таблицы (редко используется, так как таблицы обычно сохраняются до конца компиляции).

Эти операции должны быть максимально эффективными, чтобы не замедлять работу компилятора.

## 4. Способы организации таблиц

Для организации таблиц используются различные структуры данных:

- **Линейные списки**: просты в реализации, но имеют низкую производительность при больших объемах данных (поиск требует линейного времени).
- **Хеш-таблицы**: обеспечивают высокую скорость поиска и добавления за счет использования хеш-функций. Однако требуют больше памяти и усилий для реализации.
- **Деревья поиска**: могут использоваться для упорядоченного хранения данных, но менее эффективны, чем хеш-таблицы.

Выбор структуры зависит от требований к производительности и объема данных.

---

## 5. Хеш-функции

Хеш-функция преобразует ключ (например, имя идентификатора) в индекс массива, где будет храниться запись. Хорошая хеш-функция должна:

- Быть быстрой в вычислении.
- Минимизировать коллизии (ситуации, когда разные ключи дают одинаковый хеш).
- Равномерно распределять ключи по таблице.

Примеры хеш-функций:

- **Метод деления**: вычисление остатка от деления ключа на размер таблицы.
- **Полиномиальный метод**: использование полиномиального хеширования для строк.
- **Сумма ASCII-кодов символов**: простой метод для строк, но подверженный коллизиям.

---

## 6. Методы рехеширования

Рехеширование используется для разрешения коллизий в хеш-таблицах. Основные методы:

- **Линейное пробирование**: при коллизии проверяется следующая ячейка таблицы (или через фиксированный шаг).
- **Квадратичное пробирование**: при коллизии проверяются ячейки с квадратичным шагом (например, +1, +4, +9).
- **Метод цепочек**: каждая ячейка таблицы содержит список элементов с одинаковым хешем.
- **Двойное хеширование**: использование второй хеш-функции для вычисления шага при коллизии.

Выбор метода зависит от характеристик данных и требований к производительности.

---

## 7. Таблицы с вычисляемым входом

Таблицы с вычисляемым входом — это структуры, где адрес записи вычисляется на основе ключа (например, имени идентификатора). Преимущества:

- Высокая скорость доступа.
- Эффективное использование памяти.

Реализация таких таблиц основана на хеш-функциях. Для минимизации коллизий важно правильно выбрать размер таблицы (обычно простое число) и хеш-функцию. Пример применения — таблицы символов и констант, где ключом является имя идентификатора или значение константы.
